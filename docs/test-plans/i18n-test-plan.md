## 2. テストスコープ

### In Scope（テスト対象）

| カテゴリ | テスト内容 |
|---------|-----------|
| **フロントエンド UI** | 言語セレクター、全画面の翻訳表示 |
| **言語切り替え** | 手動切り替え、localStorage 永続化、ブラウザ設定検出 |
| **ICU Message Format** | パラメータ置換、複数形、動的テキスト |
| **API 通信** | Accept-Language ヘッダー送信 |
| **バックエンド API** | エラーメッセージの多言語対応 |
| **バックエンド ログ** | ログメッセージの多言語対応 |
| **フォールバック** | 未対応言語、翻訳キーなし時の挙動 |
| **アクセシビリティ** | html lang 属性、aria-label |
| **品質保証ツール** | pre-commit フック（翻訳キー整合性チェック） |
| **パフォーマンス** | 言語切り替え時間、初期ロード時間 |
| **セキュリティ** | XSS 脆弱性（パラメータインジェクション） |

### Out of Scope（テスト対象外）

| 項目 | 理由 |
|------|------|
| 翻訳の品質・正確性 | 専門翻訳者による別途レビュー |
| RTL（右から左）言語 | 対応言語に含まれない |
| サーバーサイドレンダリング | 現在クライアントサイドのみ |

### 4.3 優先度定義

#### P0（Critical）- 17 件

**定義:** i18n 機能の核となる必須テスト。不合格の場合、リリースをブロックする。

| カテゴリ | テストケース | 理由 |
|---------|-------------|------|
| **言語切り替え** | TC-FE-001, TC-FE-002, TC-FE-003, TC-FE-004 | 言語切り替え機能が動作しなければ i18n 対応が無意味 |
| **ICU 基盤** | TC-FE-005, TC-FE-011, TC-FE-012 | パラメータ置換・複数形・日付フォーマットは設計書の P0 機能 |
| **API 統合** | TC-FE-007, TC-FE-008 | フロントエンド⇔バックエンド間の言語連携必須 |
| **バックエンド基盤** | TC-BE-001, TC-BE-002, TC-BE-003, TC-BE-004, TC-BE-005 | I18nService とミドルウェアが動作しなければ API 多言語化不可 |
| **品質保証** | TC-QA-001, TC-QA-002 | pre-commit フックは翻訳キー整合性の最終防衛線 |
| **回帰** | TC-REG-001 | 既存機能への影響確認は必須 |
