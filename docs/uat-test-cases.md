## 1. テスト概要

### 1.1 テスト目的

Agent Teams Dashboardの機能が機能設計書に基づき正しく動作し、Product Ownerとエンドユーザーの期待を満たすことを検証する。

### 1.2 テスト環境

| 項目 | 要件 |
|-----|------|
| バックエンド | FastAPIサーバー (port 8000) |
| フロントエンド | Vite開発サーバー (port 5173) |
| ブラウザ | Chrome, Firefox, Safari最新版 |
| 画面サイズ | Desktop (1920×1080) ※Tablet/Mobileは対象外 |
| テストデータ | `~/.claude/` ディレクトリ内のチーム・タスクデータ |

### 1.3 優先度定義

| 優先度 | 説明 |
|-------|------|
| **P0** | リリースブロッカー。必須機能。動作しないとリリース不可。 |
| **P1** | 重要機能。主要ユーザーシナリオに影響。できる限り対応。 |
| **P2** | 改善機能。エンハンスメントやエッジケース。時間があれば対応。 |

### 1.4 テスト結果記録方法

各テストケースの末尾に以下を記録：

```
実行日: YYYY-MM-DD
テスター: [氏名]
結果: ✅ Pass / ❌ Fail / ⏭️ Skip
備考: [問題点やコメント]
```
