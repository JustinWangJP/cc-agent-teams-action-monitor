# システム使用書（ユーザーガイド）

## 1. はじめに

### 1.1 本システムの概要

Agent Teams Dashboardは、Claude CodeのAgent Teams機能をリアルタイムに監視・管理するためのダッシュボードアプリケーションです。

### 1.2 想定ユーザー

- Claude Codeを使用してAgent Teamsを運用する開発者
- チームのタスク進捗を確認したいプロジェクトマネージャー
- エージェント間の通信内容を確認したいユーザー

### 1.3 前提条件

- Claude Codeがインストールされていること
- Agent Teams機能を使用していること
- ~/.claude/ディレクトリが存在すること

### 1.4 設計思想

#### なぜHTTPポーリングか

Claude Code がファイルを直接更新するため、サーバーからの Push 通知ができません。そのため、**HTTP ポーリング** で定期的にデータを取得しています。

| 方式 | メリット | デメリット |
|------|---------|-----------|
| HTTP ポーリング（採用） | シンプルな実装、Claude Code と相性良い | リアルタイム性は劣る |
| WebSocket Push | 高いリアルタイム性 | Claude Code から通知できない |

ポーリング間隔は **5秒〜60秒** から選択可能（デフォルト30秒）。

#### チームステータス判定

チームのステータスは **セッションログの最終更新時刻（mtime）** に基づいて判定します：

| ステータス | 判定条件 | 削除ボタン |
|-----------|---------|-----------|
| **active** | セッションログ mtime ≤ 1時間 | 非表示 |
| **stopped** | セッションログ mtime > 1時間 | **表示** |
| **unknown** | セッションログなし | 非表示 |
| **inactive** | メンバーなし | 非表示 |

#### 統合タイムライン

**TimelineService** が2つのデータソースを統合：

1. **inbox メッセージ**: `~/.claude/teams/{name}/inboxes/{agent}.json`
2. **セッションログ**: `~/.claude/projects/{hash}/{sessionId}.jsonl`

これにより、エージェント間通信とユーザー対話を一元的に確認できます。

## 3. インストール手順

### 3.1 バックエンド

```bash
# 1. プロジェクトディレクトリに移動
cd cc-agent-teams-action-monitor/backend

# 2. 依存関係をインストール
pip install -e ".[dev]"

# 3. インストール確認
python -c "from app.main import app; print('OK')"
```

### 3.2 フロントエンド

```bash
# 1. プロジェクトディレクトリに移動
cd cc-agent-teams-action-monitor/frontend

# 2. 依存関係をインストール
npm install

# 3. インストール確認
npm run build
```

---

## 4. 起動手順

### 4.1 バックエンド起動

```bash
# 開発モードで起動
cd backend
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

起動成功時の出力例:
```
INFO:     Uvicorn running on http://127.0.0.1:8000
File watcher started on /Users/xxx/.claude
CacheService started (config_ttl=30s, inbox_ttl=60s)
```

### 4.2 フロントエンド起動

```bash
# 開発モードで起動
cd frontend
npm run dev
```

起動成功時の出力例:
```
VITE v5.0.0  ready in 500 ms
➜  Local:   http://localhost:5173/
```

### 4.3 アクセス方法

ブラウザで http://localhost:5173/ にアクセスしてください。

### 5.4 チームステータス表示

チームカードには現在のステータスが表示されます：

| ステータス | アイコン | 説明 |
|-----------|---------|------|
| active | 🟢 | セッションがアクティブ（1時間以内に更新） |
| stopped | 🟡 | セッション停止（1時間以上更新なし） |
| unknown | ⚪ | セッションログなし |
| inactive | ⚫ | メンバーなし |

### 5.5 エージェント状態表示

エージェントの状態は以下のロジックで推論されます：

| 状態 | アイコン | 判定条件 |
|------|---------|---------|
| idle | 💤 | 5分以上無活動 |
| working | 🔵 | in_progress タスクあり |
| waiting | ⏳ | blocked なタスクあり |
| error | ❌ | 30分以上無活動 |
| completed | ✅ | 全タスク完了 |

状態推論に使用するデータ：
- inbox メッセージ（task_assignment, task_completed 等）
- タスク定義（owner, status, blockedBy）
- セッションログ（最終活動時刻、使用モデル）

## 6. 操作方法

### 6.1 チームの選択

1. チームカードをクリック
2. 選択されたチームがハイライト表示
3. 詳細パネルにメンバー情報が表示
4. もう一度クリックで選択解除

### 6.2 タスクの確認

1. Tasksビューに切り替え
2. タスクはステータス別に表示:
   - Pending: 灰色
   - In Progress: 青色
   - Completed: 緑色
   - Stopped: 濃い灰色（24時間以上更新なし）
3. チームフィルタで特定チームのタスクのみ表示
4. 検索ボックスで件名・担当者を検索

### 6.3 メッセージタイムラインの使用

#### 時間範囲フィルタ
- プリセット: 1時間、24時間、7日間、30日間、全期間
- カスタム: スライダーで範囲を指定

#### 送信者フィルタ
- 特定エージェントのメッセージのみ表示
- 複数選択可能

#### メッセージタイプフィルタ
- message: 通常メッセージ
- task_assignment: タスク割り当て
- idle_notification: アイドル通知
- shutdown系: シャットダウン関連
- plan系: プラン承認関連

#### 検索機能
- 全文検索でメッセージ内容を検索

#### ブックマーク機能
- メッセージにマウスオーバーしてブックマークボタンをクリック
- ブラウザのローカルストレージに保存

### 6.4 チャットパネルの使用

1. チームを選択するとチャットパネルが表示
2. メッセージは時系列順に表示
3. 日付セパレーターで日付ごとに区切られる
4. Markdown形式のメッセージは整形して表示

### 6.5 チームの削除

**stopped** 状態のチームのみ削除可能です。

1. チームカードの「削除」ボタン（🗑️）をクリック
2. 確認ダイアログで「削除」を選択
3. 以下のファイルが削除されます：
   - `~/.claude/teams/{team-name}/` - チーム設定全体
   - `~/.claude/tasks/{team-name}/` - タスク定義
   - `~/.claude/projects/{project-hash}/{session}.jsonl` - セッションログ

**注意**: active 状態のチームは削除できません。

### 6.6 ポーリング間隔の調整

ヘッダーのポーリング間隔セレクターで設定：

| 間隔 | 用途 |
|------|------|
| 5秒 | リアルタイム性重視（高負荷） |
| 10秒 | バランス型 |
| 20秒 | 通常使用 |
| 30秒（推奨） | デフォルト値 |
| 60秒 | 低負荷・省電力 |

### 6.7 統合タイムラインの使用

Timelineビューでは、inbox メッセージとセッションログを統合して表示します。

#### セッションログエントリタイプ

| タイプ | 内容 | アイコン |
|--------|------|---------|
| user_message | ユーザー入力 | 👤 |
| assistant_message | アシスタント応答 | 🤖 |
| thinking | 思考プロセス | 💭 |
| tool_use | ツール呼び出し | 🔧 |
| file_change | ファイル変更 | 📁 |

#### ファイル変更履歴

`file-history-snapshot` エントリからファイル変更を表示：

| 操作 | アイコン | 色 |
|------|---------|-----|
| created | ✨ | 緑 |
| modified | ✏️ | 青 |
| deleted | 🗑️ | 赤 |
| read | 📖 | 灰 |

## 7. トラブルシューティング

### 7.1 よくある問題と解決方法

| 問題 | 原因 | 解決方法 |
|------|------|----------|
| チームが表示されない | ~/.claude/teams/が空 | Claude Codeでチームを作成 |
| データが更新されない | バックエンド停止 | バックエンドを再起動 |
| ページが読み込めない | フロントエンド未起動 | npm run devを実行 |
| API エラー | CORS設定不一致 | 設定を確認 |
| メッセージが表示されない | インボックスが空 | エージェントがメッセージを送信 |
| 更新が遅い | ポーリング間隔が長い | ヘッダーから間隔を短く設定 |
| チームが削除できない | active状態 | セッション停止後に削除可能 |

### 7.2 エラーメッセージ一覧

| エラー | 説明 | 対処法 |
|--------|------|--------|
| 404 Not Found | リソースが存在しない | データを確認 |
| 500 Internal Error | サーバーエラー | ログを確認 |

## 8. FAQ

### Q1: チームを作成するにはどうすればいいですか？

A: Claude Codeで `/teamworks` コマンドまたは `TeamCreate` ツールを使用してください。

### Q2: データが更新されません

A: 以下を確認してください:
1. バックエンドが起動している
2. ポーリング間隔が適切に設定されている
3. ポート8000がブロックされていない

### Q3: データはどこに保存されますか？

A: `~/.claude/` ディレクトリ以下のJSONファイルに保存されます。

### Q4: チームが「停止」状態と表示されます

A: **1時間以上**セッションログの更新がないチームは自動的に「停止」状態として表示されます。チームを再開すると「アクティブ」に戻ります。

### Q5: メッセージのブックマークはどこに保存されますか？

A: ブラウザのローカルストレージに保存されます。ブラウザを変えると引き継がれません。

## 9. 用語集

| 用語 | 説明 |
|------|------|
| Agent Team | Claude Codeで定義されたエージェントのグループ |
| Task | チーム内で管理される作業単位 |
| Inbox | エージェント間のメッセージ受信箱 |
| Timeline | 時系列でメッセージを表示するビュー |
| Protocol Message | エージェント間通信の定型メッセージ |
| HTTP Polling | 定期的にサーバーへリクエストを送信してデータを取得する方式 |
| FileWatcher | ファイルシステム変更を監視し、キャッシュを無効化するサービス |
| CacheService | メモリキャッシュによる高速化サービス |
| Session Log | Claude Codeのセッション履歴（.jsonlファイル） |
| mtime | ファイルの最終更新時刻（ステータス判定に使用） |
