## 1. 测试概述

### 1.1 测试目的

验证 Agent Teams Dashboard 的功能是否根据功能设计规范正常工作，并满足产品负责人和最终用户的期望。

### 1.2 测试环境

| 项目 | 要求 |
|-----|------|
| 后端 | FastAPI 服务器 (端口 8000) |
| 前端 | Vite 开发服务器 (端口 5173) |
| 浏览器 | Chrome、Firefox、Safari 最新版本 |
| 屏幕尺寸 | 桌面 (1920×1080) *平板/移动端不在范围内* |
| 测试数据 | `~/.claude/` 目录中的团队/任务数据 |

### 1.3 优先级定义

| 优先级 | 说明 |
|--------|------|
| **P0** | 发布阻塞。必需功能。不工作则无法发布。 |
| **P1** | 重要功能。影响主要用户场景。尽可能处理。 |
| **P2** | 增强功能。改进和边缘情况。有时间则处理。 |

### 1.4 测试结果记录方法

在每个测试用例末尾记录以下内容：

```
执行日期: YYYY-MM-DD
测试人员: [姓名]
结果: ✅ 通过 / ❌ 失败 / ⏭️ 跳过
备注: [问题或评论]
```

### 1.5 设计理念

#### HTTP 轮询相关测试

本系统通过 **HTTP 轮询** 实现实时更新。不需要 WebSocket 测试。

重要的测试考虑因素：
1. **轮询间隔**: 可通过页眉设置为 5 到 60 秒
2. **staleTime**: 10 秒内使用缓存
3. **refetchInterval**: 按设定间隔自动重新获取

#### 团队状态判定

状态由 **会话日志 mtime** 判定：

| 状态 | 条件 |
|------|------|
| active | 会话日志 mtime ≤ 1 小时 |
| stopped | 会话日志 mtime > 1 小时 |
| unknown | 无会话日志 |
| inactive | 无成员 |
