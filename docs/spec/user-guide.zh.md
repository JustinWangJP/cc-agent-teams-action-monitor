# 系统使用指南

**语言:** [English](user-guide.en.md) | [日本語](user-guide.md) | [中文](user-guide.zh.md)

## 1. 简介

### 1.1 系统概述

Agent Teams Dashboard 是一个用于实时监控和管理 Claude Code 的 Agent Teams 功能的仪表板应用程序。

### 1.2 目标用户

- 使用 Claude Code 运营 Agent Teams 的开发者
- 希望确认任务进度的项目经理
- 希望确认代理间通信的用户

### 1.3 前提条件

- 已安装 Claude Code
- 正在使用 Agent Teams 功能
- ~/.claude/ 目录存在

### 1.4 设计理念

#### 为什么使用 HTTP 轮询

由于 Claude Code 直接更新文件，无法从服务器发送推送通知。因此，使用 **HTTP 轮询** 定期获取数据。

| 方式 | 优点 | 缺点 |
|------|------|------|
| HTTP 轮询（采用） | 实现简单，与 Claude Code 兼容性好 | 实时性较差 |
| WebSocket 推送 | 高实时性 | 无法接收来自 Claude Code 的通知 |

轮询间隔可从 **5 秒到 60 秒** 中选择（默认 30 秒）。

#### 团队状态判定

团队状态基于 **会话日志的最后修改时间（mtime）** 进行判定：

| 状态 | 判定条件 | 删除按钮 |
|------|---------|----------|
| **active** | 会话日志 mtime ≤ 1 小时 | 隐藏 |
| **stopped** | 会话日志 mtime > 1 小时 | **显示** |
| **unknown** | 无会话日志 | **显示** |
| **inactive** | 无成员 | **显示** |

#### 统一时间线

**TimelineService** 整合两个数据源：

1. **inbox 消息**: `~/.claude/teams/{name}/inboxes/{agent}.json`
2. **会话日志**: `~/.claude/projects/{hash}/{sessionId}.jsonl`

这样可以集中查看代理间通信和用户交互。

## 3. 安装步骤

### 3.1 后端

```bash
# 1. 进入项目目录
cd cc-agent-teams-action-monitor/backend

# 2. 安装依赖
pip install -e ".[dev]"

# 3. 确认安装
python -c "from app.main import app; print('OK')"
```

### 3.2 前端

```bash
# 1. 进入项目目录
cd cc-agent-teams-action-monitor/frontend

# 2. 安装依赖
npm install

# 3. 确认安装
npm run build
```

---

## 4. 启动步骤

### 4.1 启动后端

```bash
# 以开发模式启动
cd backend
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

启动成功时的输出示例：
```
INFO:     Uvicorn running on http://127.0.0.1:8000
File watcher started on /Users/xxx/.claude
CacheService started (config_ttl=30s, inbox_ttl=60s)
```

### 4.2 启动前端

```bash
# 以开发模式启动
cd frontend
npm run dev
```

启动成功时的输出示例：
```
VITE v5.0.0  ready in 500 ms
➜  Local:   http://localhost:5173/
```

### 4.3 访问方法

在浏览器中访问 http://localhost:5173/。

## 6. 操作方法

### 6.1 选择团队

1. 点击团队卡片
2. 选中的团队会高亮显示
3. 详细信息面板显示成员信息
4. 再次点击取消选择

### 6.2 确认任务

1. 切换到 Tasks 视图
2. 任务按状态显示：
   - Pending: 灰色
   - In Progress: 蓝色
   - Completed: 绿色
   - Stopped: 深灰色（24小时以上未更新）
3. 使用团队筛选器仅显示特定团队的任务
4. 使用搜索框按主题或负责人搜索

### 6.3 使用消息时间线

#### 时间范围筛选
- 预设：1小时、24小时、7天、30天、全部
- 自定义：使用滑块指定范围

#### 发送者筛选
- 仅显示特定代理的消息
- 支持多选

#### 消息类型筛选
- message: 普通消息
- task_assignment: 任务分配
- idle_notification: 空闲通知
- shutdown*: 关闭相关
- plan*: 计划批准相关

#### 搜索功能
- 全文搜索消息内容

#### 书签功能
- 鼠标悬停在消息上并点击书签按钮
- 保存到浏览器本地存储

### 6.4 使用聊天面板

1. 选择团队后显示聊天面板
2. 消息按时间顺序显示
3. 用日期分隔符按日期分隔
4. Markdown 格式的消息会格式化显示

### 6.5 删除团队

状态为 **active 以外**（stopped / unknown / inactive）的团队可以删除。

1. 点击可删除团队卡片的"删除"按钮（🗑️）
2. 在确认对话框中选择"删除"
3. 将删除以下文件：
   - `~/.claude/teams/{team-name}/` - 整个团队配置
   - `~/.claude/tasks/{team-name}/` - 任务定义
   - `~/.claude/projects/{project-hash}/{session}.jsonl` - 会话日志

**注意**: active 状态的团队无法删除。

### 6.6 调整轮询间隔

通过标题栏的轮询间隔选择器配置：

| 间隔 | 用途 |
|------|------|
| 5 秒 | 重视实时性（高负载） |
| 10 秒 | 平衡型 |
| 20 秒 | 正常使用 |
| 30 秒（推荐） | 默认值 |
| 60 秒 | 低负载、省电 |

### 6.7 使用统一时间线

Timeline 视图整合显示 inbox 消息和会话日志。

#### 会话日志条目类型

| 类型 | 内容 | 图标 |

| user_message | 用户输入 | 👤 |
| assistant_message | 助手响应 | 🤖 |
| thinking | 思考过程 | 💭 |
| tool_use | 工具调用 | 🔧 |

## 7. 故障排除

### 7.1 常见问题和解决方法

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 团队不显示 | ~/.claude/teams/ 为空 | 在 Claude Code 中创建团队 |
| 数据不更新 | 后端停止 | 重启后端 |
| 页面无法加载 | 前端未启动 | 运行 npm run dev |
| API 错误 | CORS 配置不匹配 | 检查配置 |
| 消息不显示 | inbox 为空 | 代理发送消息 |
| 更新缓慢 | 轮询间隔过长 | 从标题栏缩短间隔 |
| 团队无法删除 | active 状态 | 会话停止后可删除 |

### 7.2 错误消息列表

| 错误 | 说明 | 处理方法 |
|------|------|----------|
| 404 Not Found | 资源不存在 | 检查数据 |
| 500 Internal Error | 服务器错误 | 检查日志 |

## 8. FAQ

### Q1: 如何创建团队？

A: 请在 Claude Code 中使用 `/teamworks` 命令或 `TeamCreate` 工具。

### Q2: 数据不更新

A: 请检查以下项目：
1. 后端正在运行
2. 轮询间隔配置适当
3. 端口 8000 未被阻止

### Q3: 数据保存在哪里？

A: 保存在 `~/.claude/` 目录下的 JSON 文件中。

### Q4: 团队显示为"停止"状态

A: **1小时以上**没有会话日志更新的团队会自动显示为"停止"状态。恢复团队后会变为"活跃"。

### Q5: 消息书签保存在哪里？

A: 保存在浏览器本地存储中。更换浏览器时无法继承。
